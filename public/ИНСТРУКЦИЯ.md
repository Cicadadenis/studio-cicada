# Cicada Studio - Руководство пользователя

## Обзор

**Cicada Studio** — визуальный конструктор Telegram-ботов на основе ReactFlow. Боты создаются путём соединения блоков на холсте, после чего конструктор генерирует код на языке Cicada (.ccd).

**Интерфейс:**
- **Левая панель** — палитра блоков (перетащите на холст)
- **Центральная область** — холст для построения схемы
- **Правая панель** — свойства выбранного блока + сгенерированный код

## Группы блоков

### 🔧 Настройки (Settings)

Блоки конфигурации бота — должны быть в начале файла.

| Блок | Иконка | Назначение | Когда использовать |
|------|--------|-----------|-------------------|
| **Версия** | 📌 | Указывает версию бота | Всегда в начале файла |
| **Бот** | 🤖 | Токен Telegram-бота | Обязательно для работы бота |
| **Команды меню** | 📋 | Команды в меню Telegram | Для отображения команд при вводе / |
| **Глобальная** | 🌍 | Глобальные переменные | Данные доступные всем пользователям |

**DSL вывод:**
```ccd
версия "1.0"
бот "YOUR_TOKEN"
команды:
    "/start" - "Запуск бота"
глобально магазин_открыт = истина
```

### 🧱 Блоки (Blocks)

Переиспользуемые фрагменты кода.

| Блок | Иконка | Назначение | Когда использовать |
|------|--------|-----------|-------------------|
| **Блок** | 🧱 | Определяет переиспользуемый блок | Когда один код нужен в нескольких местах |
| **Использовать** | ⚡ | Вызов ранее созданного блока | Для вставки блока в поток |

**Важно:** Блок «Использовать» должен ссылаться на имя блока, определённого через блок «Блок».

**DSL вывод:**
```ccd
блок приветствие:
    ответ "Привет!"
    
использовать приветствие
```

### ⚙️ Middleware

Код, выполняемый при каждом сообщении.

| Блок | Иконка | Назначение | Когда использовать |
|------|--------|-----------|-------------------|
| **Middleware** | ⚙ | Код до/после каждого сообщения | Логирование, проверки, модификация |

**Параметры:**
- `type`: `before` (до обработки) или `after` (после)
- `return`: `true` — прервать дальнейшую обработку

**DSL вывод:**
```ccd
до каждого:
    лог "[{чат.id}] {пользователь.имя}: {текст}"
```

### ▶️ Основные (Core)

Основные обработчики событий.

| Блок | Иконка | Назначение | Когда использовать |
|------|--------|-----------|-------------------|
| **Старт** | ▶ | Точка входа при `/start` | Главное меню, приветствие |
| **Ответ** | ✉ | Отправка текстового сообщения | Любой текстовый вывод |
| **Кнопки** | ⊞ | Клавиатура с кнопками | Навигация, выбор вариантов |
| **Команда** | / | Обработка команды `/cmd` | Специальные команды бота |
| **Нажатие** | ⊙ | Обработка нажатия кнопки | Callback от inline-кнопок |

**DSL вывод:**
```ccd
старт:
    ответ "👋 Привет!"
    кнопки "🛍️ Магазин, 👤 Профиль"

команда "/help":
    ответ "Справка по боту"

при нажатии "🛍️ Магазин":
    ответ "Вы в магазине"
```

### 🧠 Логика (Logic)

Управление потоком и данными.

| Блок | Иконка | Назначение | Когда использовать |
|------|--------|-----------|-------------------|
| **Условие** | ◇ | If-else ветвление | Проверка условий |
| **Спросить** | ? | Запрос ввода от пользователя | Сбор данных |
| **Запомнить** | ♦ | Сохранить в переменную | Временное хранение |
| **Получить** | 📥 | Загрузить из хранилища | Чтение сохранённых данных |
| **Сохранить** | 💾 | Сохранить в хранилище | Постоянное хранение |
| **Рандом** | ⚄ | Случайный выбор | Разнообразие ответов |
| **Переключатель** | ⇄ | Switch-case по переменной | Множественное ветвление |
| **Цикл** | ↻ | Повторение блоков | Итерации, while/count |

**DSL вывод:**
```ccd
если текст == "да":
    ответ "Отлично!"
иначе:
    ответ "Жаль"

спросить "Как вас зовут?" → имя
запомни счёт = 0
получить "посещений" → счётчик
сохранить "посещений" = счётчик

рандом:
    "Привет!"
    "Здорово!"
    
переключить текст:
    "да":
        ответ "Согласен"
    "нет":
        ответ "Не согласен"

повторять 3 раз:
    ответ "Итерация"

пока balance > 0:
    ответ "Ещё есть деньги"
```

### 🚀 Действия (Actions)

Вспомогательные операции.

| Блок | Иконка | Назначение | Когда использовать |
|------|--------|-----------|-------------------|
| **Пауза** | ⏱ | Задержка перед следующим блоком | Имитация "думает" |

| **HTTP-запрос** | ↗ | API запросы | Интеграция с внешними сервисами |
| **Переход** | → | Переход к сценарию | Навигация между сценариями |
| **Стоп** | ■ | Завершение диалога | Прерывание цепочки |
| **База данных** | 🗄 | SQL запросы | Работа с БД |
| **Классификация** | 🧠 | AI-определение намерения | NLP, бот понимает смысл |
| **Лог** | 📋 | Запись в лог | Отладка, мониторинг |
| **Проверка роли** | 🔐 | Ветвление по роли | Admin/User/Gate |

**DSL вывод:**
```ccd
пауза 2с

лог[info] "Пользователь нажал кнопку"

    "admin":
        ответ "Панель администратора"
```

### 🔄 Сценарии (Scenarios)

Многошаговые сценарии.

| Блок | Иконка | Назначение | Когда использовать |
|------|--------|-----------|-------------------|
| **Сценарий** | ↺ | Определение сценария | Многошаговые процессы |
| **Шаг** | » | Шаг внутри сценария | Этапы сценария |
| **Inline-кнопки** | ▦ | Inline клавиатура | Кнопки под сообщением |

**DSL вывод:**
```ccd
сценарий регистрация:
    шаг начало:
        ответ "Начинаем регистрацию!"
    шаг имя:
        ответ "Введите имя"

inline-кнопки:
    ["Да" → "callback_да", "Нет" → "callback_нет"]
```

#### ⚠️ Частые ошибки ИИ / антипаттерны

1. **Не ставьте `стоп` сразу после `запустить имя_сценария` в том же обработчике** (`старт`, `при нажатии`, команда и т.д.). В ядре Cicada это добавляет `EndScenario` в общую очередь вместе с отложенными шагами после `спросить` — сценарий обрывается или ведёт себя непредсказуемо. Цепочки вида «ответ → запустить оформление» должны **заканчиваться на `запустить` без `стоп`**. Завершение диалога (`стоп`) — внутри самого сценария, когда шаги действительно закончились.

2. **Подряд идущие шаги сценария** (`шаг а:` … `шаг б:`) выполняются **по порядку автоматически**. Не нужно после каждого `спросить` вставлять `перейти "шаг_б"` — это усложняет граф и очередь отложенных инструкций. Блок «Переход» имеет смысл для ветвления (`если` → разные шаги) или перехода на другой сценарий.

> В панели «Проверить» Cicada Studio опасная связка `запустить` + следующая строка `стоп` подсвечивается и **автоматически убирается** из `.ccd`.

## Как соединять блоки

### Правила соединений:

1. **От source (правый handle) к target (левый handle)** — поток выполнения идёт слева направо

2. **Корневые блоки** (начало цепочки):
   - `start` — обработка /start
   - `command` — обработка /команды
   - `scenario` — начало сценария
   - `callback` — обработка нажатия

3. **Блоки-контейнеры** (могут иметь вложенные блоки):
   - `block` — вложенные блоки = тело блока
   - `middleware` — вложенные = код middleware
   - `condition` — вложенные = ветка "если"
   - `loop` — вложенные = тело цикла
   - `scenario` — шаги сценария

4. **Завершающие блоки** (обычно не имеют исходящих):
   - `stop` — остановка
   - `goto` — переход (не продолжает текущую цепочку)

### Примеры схем:

**Простое приветствие:**
```
[start] → [typing] → [message] → [buttons]
```

**Обработка команды:**
```
[command /help] → [message]
```

**Условная логика:**
```
[ask] → [condition] → [message "Да"]
              ↓
         [message "Нет"]
```

**С использованием блоков:**
```
[block "приветствие"] → [message] → [buttons]

[start] → [use "приветствие"] → [message "Продолжение..."]
```

## Переменные и подстановки

В текстовых полях можно использовать переменные:

| Синтаксис | Значение |
|-----------|----------|
| `{пользователь.имя}` | Имя пользователя |

| `{чат.id}` | ID чата |

**Пример:**
```
Привет, {пользователь.имя}! Вы посещали нас {счётчик} раз.
```

## Пример простого бота

```ccd
версия "1.0"
бот "YOUR_BOT_TOKEN"
команды:
    "/start" - "🚀 Запуск бота"
    "/help" - "❓ Помощь"

старт:
    ответ "👋 Привет, {пользователь.имя}!"
    кнопки "🛍️ Магазин, 👤 Профиль"

при нажатии "🛍️ Магазин":
    ответ "Добро пожаловать в магазин!"
    
команда "/help":
    ответ "📚 Доступные команды:\n/start - главное меню\n/help - помощь"
```

**Схема:**
1. Блок «Версия» → Блок «Бот» → Блок «Команды меню» (настройки)
2. Блок «Старт» → Блок «Ответ» → Блок «Кнопки» (главное меню)
3. Блок «Нажатие» (🛍️ Магазин) → Блок «Ответ»
4. Блок «Команда» (/help) → Блок «Ответ»
